## Notes & caveats

- **`clock_in` / `clock_out` are deliberately omitted** from the MCP surface as advertised to chat clients in some deployments. The functions exist in [`apps/mcp_server/tools/timecards.py`](../apps/mcp_server/tools/timecards.py) but you may see them filtered out by deploy-time policy. Audit your deployment's tool-allow list before assuming a tool is available.
- **Description quality is uneven across modules.** `content_packs`, `sprite_authoring`, and the sprite playbook were authored with LLM-as-client in mind; some older modules lean on terse docstrings that don't disambiguate from their neighbors. The four chronicle/creations neighbors (`write_journal`, `update_journal`, `create_manual_entry`, `log_creation`, `list_chronicle_entries`) were sharpened in the same PR that produced this doc.
- **Empty-state hint on `list_templates`.** A fresh-ish family has no templates. The onramp is `save_project_as_template` from an existing project â€” there is no seed pack today.
- **Overlap-by-design between `chronicle` and `creations`.** Logging a creation always emits a Chronicle `CREATION` entry â€” these are not redundant tools; see the docstrings for the distinction.
